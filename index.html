Bitonic Sorting Network Teaching Tool

A clean, step-by-step visual tool for learning data-oblivious sorting networks and bitonic merge behavior.

Overview

Bitonic sort is a sorting network: a fixed sequence of compare-and-swap operations that does not depend on the actual input values. It works by building bitonic sequences, then merging them with the same network pattern for every array of the same size.

  • Fixed network: comparator positions are defined before runtime.
  • Bitonic sequence: a subsequence that rises then falls, or can be rotated to that shape.
  • Data-oblivious: the same comparator structure works for all inputs of a given length.

Explanation

A bitonic sequence is built from two sorted halves: one ascending and one descending. Bitonic sort recursively creates these halves so that the combined block forms a bitonic pattern before merge.

Bitonic merge then compares elements at fixed distances across the midpoint and moves larger values one way and smaller values the other. Every compare-exchange step is predetermined, which is why this method is useful for teaching parallel network behavior.

  1. Build: sort the first half upward and the second half downward.
  2. Merge: compare mirrored positions across the midpoint in the target direction.
  3. Repeat: recursively merge smaller bitonic segments until the block is ordered.

Pseudocode

1  procedure bitonicSort(lo, n, dir)
2      if n <= 1 return
3      k := n / 2
4      bitonicSort(lo, k, ASC)
5      bitonicSort(lo + k, k, DESC)
6      bitonicMerge(lo, n, dir)

7  procedure bitonicMerge(lo, n, dir)
8      if n <= 1 return
9      k := n / 2
10     for i from lo to lo + n - k - 1
11         compareAndSwap(i, i + k, dir)
12     bitonicMerge(lo, k, dir)
13     bitonicMerge(lo + k, k, dir)

14 procedure compareAndSwap(i, j, dir)
15     if (dir = ASC and value[i] > value[j]) or
16        (dir = DESC and value[i] < value[j]) then
17         swap value[i] and value[j]
dir = ASC or DESC · lo = starting index · n = length of subarray

Worked example

Example input: [3, 7, 4, 8, 6, 2, 1, 5]. The network first arranges two halves in opposite directions, then merges them into order.

Input: [3, 7, 4, 8, 6, 2, 1, 5]
Build bitonic halves: left half upward [3, 4, 7, 8], right half downward [6, 5, 2, 1]
Bitonic merge: compare fixed cross-block pairs and recursively reorder into one sorted block.
Final sorted order: [1, 2, 3, 4, 5, 6, 7, 8]
