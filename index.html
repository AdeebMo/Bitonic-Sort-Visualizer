Overview

What bitonic sort is doing

Bitonic sort is a sorting network: it follows a predetermined pattern of comparators instead of reacting to the values at runtime.

  • Fixed structure: the comparator layout is known before the algorithm starts.
  • Bitonic sequences: each block rises then falls, or can be rotated into that shape.
  • Parallel-friendly: passes are easier to reason about as a network than as ordinary branching code.
Flow

How the merge pattern works

The algorithm first creates opposing sorted halves, then repeatedly compares elements at fixed distances until the whole block is ordered in the target direction.

  1. Build: sort the left half ascending and the right half descending.
  2. Compare: execute mirrored compare-and-swap pairs across the midpoint.
  3. Refine: recurse into smaller bitonic segments until the final order emerges.
Pseudocode 
1  procedure bitonicSort(lo, n, dir)
2      if n <= 1 return
3      k := n / 2
4      bitonicSort(lo, k, ASC)
5      bitonicSort(lo + k, k, DESC)
6      bitonicMerge(lo, n, dir)

7  procedure bitonicMerge(lo, n, dir)
8      if n <= 1 return
9      k := n / 2
10     for i from lo to lo + n - k - 1
11         compareAndSwap(i, i + k, dir)
12     bitonicMerge(lo, k, dir)
13     bitonicMerge(lo + k, k, dir)
dir chooses ascending or descending order. lo and n define the active segment.
Example

One worked sequence

Starting from [3, 7, 4, 8, 6, 2, 1, 5], bitonic sort builds opposing halves before running a predictable merge schedule.

Input: [3, 7, 4, 8, 6, 2, 1, 5]
Build: left side trends upward, right side trends downward.
Merge: fixed-distance comparisons move smaller values toward the target side.
Finish: the network resolves to [1, 2, 3, 4, 5, 6, 7, 8].
